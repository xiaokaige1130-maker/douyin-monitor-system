# 抖音监控系统

24小时监控抖音同行开播时间、开播时长、直播间数据的全自动系统。

## 功能特性

- ✅ **24小时自动监控**：无需人工干预，全天候监控
- ✅ **开播即时检测**：实时检测同行开播状态
- ✅ **直播间数据采集**：在线人数、点赞、评论、分享等数据
- ✅ **时长统计**：精确记录开播时长和峰值时间
- ✅ **多账号并发监控**：支持同时监控多个抖音账号
- ✅ **智能调度**：开播期高频监控，非开播期低频监控
- ✅ **实时通知**：开播/下播即时通知（钉钉、微信、邮件）
- ✅ **数据可视化**：Grafana仪表板展示数据趋势
- ✅ **API接口**：提供RESTful API查询数据
- ✅ **数据持久化**：PostgreSQL + InfluxDB存储历史数据

## 快速开始

### 1. 环境要求

- Docker & Docker Compose
- Python 3.8+
- 至少4GB内存

### 2. 安装步骤

```bash
# 克隆项目
git clone <repository-url>
cd douyin_monitor

# 复制环境配置文件
cp .env.example .env

# 编辑配置文件
vim .env  # 配置您的设置

# 启动系统
chmod +x start.sh restart.sh stop.sh
./start.sh
```

### 3. 配置说明

编辑 `.env` 文件配置以下内容：

- **抖音Cookie/Token**：用于API访问（可选）
- **通知配置**：钉钉、微信、邮件通知
- **代理配置**：如果需要使用代理
- **监控账号**：在系统启动后通过API添加

### 4. 访问服务

系统启动后可以访问：

- **监控系统API**: http://localhost:8000
- **Grafana仪表板**: http://localhost:3000 (admin/admin123)
- **InfluxDB UI**: http://localhost:8086 (admin/admin123)

## 使用方法

### 添加监控账号

```bash
# 通过API添加账号
curl -X POST "http://localhost:8000/api/accounts?douyin_id=目标抖音ID&nickname=账号昵称"
```

### 查看监控状态

```bash
# 查看系统状态
curl "http://localhost:8000/api/system/status"

# 查看活跃直播
curl "http://localhost:8000/api/lives/active"

# 查看监控账号
curl "http://localhost:8000/api/accounts"
```

### 数据查询

系统提供丰富的API接口：

- `GET /api/accounts` - 获取监控账号列表
- `GET /api/lives/active` - 获取活跃直播
- `GET /api/lives/recent` - 获取最近直播
- `GET /api/stats/daily` - 获取每日统计
- `GET /api/stats/account/{id}` - 获取账号统计

## 数据采集方式

系统支持多种数据采集方式，按稳定性排序：

### 1. 抖音官方API（最稳定）
- 需要有效的Cookie/Token
- 请求频率受限
- 数据最准确

### 2. 抖音Web API
- 通过分析网页接口
- 需要定期维护
- 中等稳定性

### 3. 浏览器模拟（最灵活）
- 使用Playwright/Selenium
- 绕过部分限制
- 速度较慢但稳定

## 通知配置

系统支持多种通知方式：

### 钉钉机器人
1. 在钉钉群添加自定义机器人
2. 获取Webhook地址
3. 配置到 `.env` 文件

### 企业微信机器人
1. 在企业微信群添加机器人
2. 获取Webhook地址
3. 配置到 `.env` 文件

### 邮件通知
1. 配置SMTP服务器信息
2. 设置发件人账号
3. 添加收件人列表

## 数据存储

### PostgreSQL
- 存储结构化数据

- 支持复杂查询和统计

### InfluxDB
- 存储时间序列数据

### Redis

- 任务调度
- 会话状态管理

## 故障排除

### 常见问题

1. **抖音API访问受限**
   - 检查Cookie/Token是否有效
   - 降低请求频率
   - 使用代理IP

2. **数据采集失败**
   - 检查网络连接
   - 验证抖音账号ID是否正确
   - 查看错误日志

3. **通知发送失败**
   - 检查通知配置
   - 验证Webhook地址
   - 查看SMTP配置

### 查看日志

```bash
# Docker服务日志
docker-compose logs -f

# 应用日志
tail -f logs/app_$(date +%Y-%m-%d).log

# 监控日志（数据库）

```

## 性能优化

### 监控大量账号
1. 调整 `MAX_CONCURRENT_CHECKS` 参数
2. 使用代理IP池
3. 合理设置检查间隔

### 数据存储优化
1. 定期清理旧数据
2. 使用数据库索引
3. 分区存储时间序列数据

## 开发指南

### 项目结构
```

├── core/           # 核心模块
│   ├── config.py   # 配置管理
│   ├── database.py # 数据库操作
│   ├── monitor.py  # 监控逻辑
│   ├── scheduler.py # 任务调度
│   └── notifier.py # 通知发送
├── api/            # API服务器

├── logs/           # 日志目录
```

1. **新增数据源**：修改 `core/monitor.py` 中的采集逻辑
2. **新增通知渠道**：在 `core/notifier.py` 中添加新方法
3. **自定义报表**：修改API接口或添加新的分析模块
